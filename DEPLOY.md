# Полная инструкция по деплою на чистый VDS

> Ubuntu 22.04 LTS. Все команды выполнять под root или через sudo.

---

## 1. Первичная настройка сервера

```bash
apt update && apt upgrade -y
apt install -y git curl wget nano ufw
```

### Настройка файрвола

```bash
ufw allow 22      # SSH
ufw allow 80      # HTTP
ufw allow 443     # HTTPS
ufw enable
```

## 3. Установка PostgreSQL 15

```bash
apt install -y postgresql postgresql-contrib

# Запуск и автостарт
systemctl enable postgresql
systemctl start postgresql

# Проверка
systemctl status postgresql
```

### Создание базы данных и пользователя

```bash
# Заходим под postgres
sudo -u postgres psql

# Внутри psql — выполни эти команды:
CREATE USER tgadmin WITH PASSWORD 'ВАШ_ПАРОЛЬ_СЮДА';
CREATE DATABASE telegram_automation OWNER tgadmin;
GRANT ALL PRIVILEGES ON DATABASE telegram_automation TO tgadmin;
\q
```

> Запомни: `tgadmin` / `ВАШ_ПАРОЛЬ_СЮДА` — это пойдёт в `.env` как DATABASE_URL

## 11. Настройка Nginx + домен

### Привязка домена

1. В панели управления доменом (Cloudflare / reg.ru / namecheap и т.д.)
2. Добавь A-запись: `@` → `IP адрес твоего VDS`
3. Добавь A-запись: `www` → `IP адрес твоего VDS`
4. Подожди 5-30 минут до распространения DNS

### Конфиг Nginx

```bash
nano /etc/nginx/sites-available/telegram-automation
```

Вставь:

```nginx
server {
    listen 80;
    server_name твой-домен.ru www.твой-домен.ru;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
        client_max_body_size 20m;
    }
}
```

```bash
# Активируем конфиг
ln -s /etc/nginx/sites-available/telegram-automation /etc/nginx/sites-enabled/

# Проверяем синтаксис
nginx -t

# Перезапускаем
systemctl reload nginx
```

### Получение SSL сертификата

```bash
certbot --nginx -d твой-домен.ru -d www.твой-домен.ru
# Следуй инструкциям: введи email, соглашайся с условиями
# Certbot автоматически обновит конфиг Nginx для HTTPS
```

Проверь автообновление:

```bash
certbot renew --dry-run
```

## 12. Настройка Telegram бота

### Создание бота у @BotFather

1. Открой Telegram, найди `@BotFather`
2. Отправь `/newbot`
3. Придумай имя бота (например: `Охота за технологиями`)
4. Придумай username бота (должен заканчиваться на `bot`, например: `ohotabot`)
5. Скопируй токен вида `1234567890:AABBCCddEEFFggHHiiJJkkLLmmNN` → вставь в `.env` как `TELEGRAM_BOT_TOKEN`

### Получение своего Telegram ID

1. Найди бота `@userinfobot`
2. Отправь ему `/start`
3. Скопируй число из поля `Id:` → вставь в `.env` как `ADMIN_TELEGRAM_IDS`

### Добавление бота в канал как администратора

**Это обязательно, иначе бот не сможет публиковать посты!**

1. Открой свой Telegram канал
2. Настройки канала → Администраторы → Добавить администратора
3. Найди своего бота по username
4. Включи права: **Публикация сообщений** ✓
5. Сохрани

### Настройка канала публикации через веб-интерфейс

После деплоя:

1. Открой `https://твой-домен.ru`
2. Зайди в **Настройки**
3. В поле **"Целевой канал"** укажи `@username_канала` или числовой ID канала
4. Сохрани настройки

> Числовой ID канала можно узнать через `@getidsbot` — перешли ему любое сообщение из канала.

## 16. Структура .env — что куда идёт

| Переменная | Откуда взять | Назначение |
|---|---|---|
| `DATABASE_URL` | Сам придумываешь при создании БД | Подключение к PostgreSQL |
| `TELEGRAM_BOT_TOKEN` | `@BotFather` → `/newbot` | Авторизация бота |
| `TELEGRAM_CHANNEL_ID` | `@username` канала или числовой ID | Канал по умолчанию для публикации |
| `OPENROUTER_API_KEY` | [openrouter.ai/keys](https://openrouter.ai/keys) | AI для генерации текстов и изображений |
| `ADMIN_TELEGRAM_IDS` | `@userinfobot` | Кто имеет доступ к боту и панели |
| `SKIP_TELEGRAM_AUTH` | `false` на проде, `true` при разработке | Пропуск авторизации через Telegram |
| `NEXT_PUBLIC_APP_URL` | Твой домен с `https://` | URL панели управления в кнопках бота |

## 17. Типичные проблемы и решения

### Бот не публикует в канал
- Убедись что бот добавлен как **администратор** канала с правом публикации
- Проверь `TELEGRAM_CHANNEL_ID` — должен быть `@username` (с @) или числовой ID вида `-1001234567890`
- В настройках сайта (раздел "Настройки") проверь поле "Целевой канал"

### Сайт открывается по IP но не по домену
- Проверь DNS: `nslookup твой-домен.ru` — должен показывать IP сервера
- Проверь Nginx: `nginx -t` и `systemctl status nginx`

### PM2 процесс падает
```bash
pm2 logs tg-automation-bot --lines 50
```
Чаще всего причина в неправильном `.env` — проверь все переменные.

### Ошибка подключения к БД
```bash
# Проверь что PostgreSQL работает
systemctl status postgresql

# Проверь что пользователь и БД созданы
sudo -u postgres psql -c "\l"
sudo -u postgres psql -c "\du"

# Проверь connection string в .env
cat /var/www/telegram-automation/.env | grep DATABASE_URL
```

### Sharp не работает (ошибки обложек)
```bash
cd /var/www/telegram-automation
bun remove sharp
bun add sharp
```
